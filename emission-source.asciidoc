== Emission Sources
include::stylesheets/Toggle[]

[[sec.emission-source.intro]]
=== Introduction

The emission-source model is used when particles are not injected all at
once at the start of a simulation, but are generated by one or more
sources during tracking.

In _OPALX_, the relevant objects are:

* `DISTRIBUTION`, which defines how particles are sampled,
* `EMISSIONSOURCE`, which binds one source to one distribution and adds
  source-specific offsets and timing,
* `EMISSIONSOURCELIST`, which groups one or more sources for attachment to
  a `BEAM` through `BEAM::SOURCES`.

The term _injection_ is used here for a distribution that is introduced
at one instant, whereas _emission_ refers to particles that are created over
multiple time steps. Time-dependent samplers such as `TYPE = FLATTOP`
belong to the latter category.

[[sec.emission-source.setup]]
=== High-Level Setup

To use emission sources in _OPALX_, proceed as follows:

1. Define one or more `DISTRIBUTION` objects.
2. For each distribution, define an `EMISSIONSOURCE` and specify any
   required source offsets `R0*`, momentum offsets `P0*`, and start time
   `T0`.
3. Collect the source names into an `EMISSIONSOURCELIST`.
4. Define the `BEAM` after the source list and attach it with
   `SOURCES = mySources`.
5. Select that beam, or several beams, on `TRACK` through `BEAM` or `BEAMS`.

The statistical properties of the beam continue to come from the linked
distribution. The source adds only source-specific placement and timing.

[[sec.emission-source.legacy]]
=== Adapting Legacy Input Files

Older inputs sometimes attached a distribution directly to `RUN`. In the
emission-source workflow, that setup must be moved into explicit source and
beam objects by wrapping the distribution in an `EMISSIONSOURCE`.

Conceptually, the migration is:

* define or keep the original `DISTRIBUTION`,
* wrap it in an `EMISSIONSOURCE`,
* add that source to an `EMISSIONSOURCELIST`,
* move `SOURCES = mySources` onto the `BEAM` definition,
* remove `BEAM = ...` from `RUN`.

In this workflow, `RUN` does not need to specify a `DISTRIBUTION`.

[[sec.emission-source.distribution]]
=== Distribution Attributes Relevant for Emission

Emission sources reuse the ordinary `DISTRIBUTION` infrastructure. Two
distribution attributes are particularly relevant for emission:

* `EMITTED`: marks the distribution as an emitted beam rather than a
  purely injected bunch. This is especially important for time-dependent
  samplers such as `TYPE = FLATTOP`.
* `NPARTDIST`: macroparticle count associated with that distribution for
  sampling.

When multiple emission sources are used, the total particle count is
derived from the per-distribution `NPARTDIST` values and checked against
`BEAM::NPART`. Both quantities remain necessary because `BEAM::NPART`
enters other derived beam quantities such as macrocharge and macromass.

For the full set of distribution parameters, see Chapter
<<chp.distribution,Distribution Command>>.

[[sec.emission-source.statement]]
=== The `EMISSIONSOURCE` Statement

An `EMISSIONSOURCE` is defined as:

----
ES1: EMISSIONSOURCE,
     DISTRIBUTION = Dist1,
     R0X = 0.0, R0Y = 0.0, R0Z = 0.0,
     P0X = 0.0, P0Y = 0.0, P0Z = 0.0,
     T0  = 0.0;
----

[[tab.emission-source.attributes,Table {counter:tab-cnt}]]
.Definition of `EMISSIONSOURCE` attributes.
[cols="<2,^1,<4",options="header"]
|=======================================================================
| Attribute Name | Units | Description

| `DISTRIBUTION` | none | Name of an existing `DISTRIBUTION` object to sample from.
| `R0X` | m | Initial source offset in latexmath:[x].
| `R0Y` | m | Initial source offset in latexmath:[y].
| `R0Z` | m | Initial source offset in latexmath:[z].
| `P0X` | latexmath:[\beta\gamma] | Initial source momentum offset in latexmath:[x].
| `P0Y` | latexmath:[\beta\gamma] | Initial source momentum offset in latexmath:[y].
| `P0Z` | latexmath:[\beta\gamma] | Initial source momentum offset in latexmath:[z].
| `T0` | s | Start time of the source. Sampling or emission begins once the tracker time reaches this value.
|=======================================================================

All offsets default to `0.0`, and `T0` defaults to `0.0`.

The source augments, but does not replace, the linked distribution. More precisely:

- `R0X`, `R0Y`, and `R0Z` are applied as source offsets on top of the
  sampled particle coordinates.
- `P0X`, `P0Y`, and `P0Z` are applied as source momentum offsets on top
  of the sampled particle momenta.
- `T0` is interpreted in simulation time. A source with `T0 > 0` is
  inactive until that time is reached.

In particular, `P0Z` is added on top of the reference longitudinal
momentum determined by the beam definition.

[[sec.emission-source.list]]
=== The `EMISSIONSOURCELIST` Statement

An `EMISSIONSOURCELIST` groups multiple emission sources into one named
object that can be referenced by `BEAM::SOURCES`:

----
mySources: EMISSIONSOURCELIST = (ES1, ES2, ES3);
----

Every list entry must be the name of a previously defined
`EMISSIONSOURCE`.

[[sec.emission-source.track]]
=== Use With `BEAM` and `TRACK`

The source list is attached to the beam:

----
myBeam: BEAM, ..., SOURCES = mySources;
----

The beam is then selected on `TRACK`:

----
TRACK, LINE = myLine, BEAM = myBeam;
----

In the current _OPALX_ implementation, simulations that use emission or
injection sources must attach an `EMISSIONSOURCELIST` to each participating
beam through `BEAM::SOURCES`.

`TRACK::SOURCES` is retained only for transitional backward compatibility and is
ignored for source resolution. New inputs should not use it.

[[sec.emission-source.examples]]
=== Examples

==== `FLATTOP` Emission with a Delayed `GAUSS` Injector

The following pattern is representative of the regression tests:

----
Dist1: DISTRIBUTION, TYPE = FLATTOP,
       EMITTED = true,
       NPARTDIST = n_particles;

ES1: EMISSIONSOURCE, DISTRIBUTION = Dist1,
     R0Z = 0.0,
     P0Z = 0.0;

DistGauss: DISTRIBUTION, TYPE = GAUSS,
           NPARTDIST = 20000;

ESGauss: EMISSIONSOURCE, DISTRIBUTION = DistGauss,
         R0Z = 0.1,
         P0Z = 0.0,
         T0 = 4 * 1e-9;

mySources: EMISSIONSOURCELIST = (ES1, ESGauss);

BEAM1: BEAM, PARTICLE = ELECTRON, PC = P0, NPART = n_particles,
       BFREQ = rf_freq, BCURRENT = beam_bunch_charge * rf_freq * 1e6,
       CHARGE = -1, SOURCES = mySources;

TRACK, LINE = myLine, BEAM = BEAM1,
       MAXSTEPS = 200, DT = {5e-11}, ZSTOP = 2.0;
RUN, METHOD = "PARALLEL", FIELDSOLVER = FS1; ENDTRACK;
----

Important points:

* `FLATTOP` is time-dependent and therefore typically uses
  `EMITTED = true`.
* A `GAUSS` distribution can be used as a delayed injector by assigning
  `T0 > 0` in the corresponding `EMISSIONSOURCE`.

==== Two Counter-Propagating `FLATTOP` Sources

----
DistL: DISTRIBUTION, TYPE = FLATTOP,
       EMITTED = true,
       NPARTDIST = 16384 * 2;

DistR: DISTRIBUTION, TYPE = FLATTOP,
       EMITTED = true,
       NPARTDIST = 16384 * 2;

ESL: EMISSIONSOURCE, DISTRIBUTION = DistL,
     R0Z = -0.016,
     P0Z = 0.4;

ESR: EMISSIONSOURCE, DISTRIBUTION = DistR,
     R0Z = 0.016,
     P0Z = -0.4;

mySources: EMISSIONSOURCELIST = (ESL, ESR);

BEAM1: BEAM, PARTICLE = ELECTRON, PC = P0, NPART = n_particles,
       BFREQ = rf_freq, BCURRENT = beam_bunch_charge * rf_freq * 1e6,
       CHARGE = -1, SOURCES = mySources;

TRACK, LINE = myLine, BEAM = BEAM1,
       MAXSTEPS = 300, DT = {dt}, ZSTOP = 1.0;
RUN, METHOD = "PARALLEL", FIELDSOLVER = FS1; ENDTRACK;
----

[[sec.emission-source.impl]]
=== Current _OPALX_ Implementation Notes

During setup, _OPALX_:

* resolves the `EMISSIONSOURCELIST` referenced by each selected `BEAM`,
* resolves the distributions referenced by each `EMISSIONSOURCE`,
* builds sampler objects for the supported distribution types,
* applies source-specific offsets `R0` and `P0`,
* activates delayed or time-dependent sources according to `T0`.

The source list therefore defines _when_ and _from where_ particles are
created, while the statistical properties of the emitted beam continue to
be determined by the linked distributions.

As of PR 315, _OPALX_ also accepts multiple beams on `TRACK` through
`BEAMS = { ... }` and allocates one particle container per selected beam during
initialisation. The actual tracking loop and field-solver coupling across all
containers are planned follow-up work.
